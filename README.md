# AI Revenue Insights Assistant

An end-to-end data and AI project that combines SQL analytics, business intelligence dashboards, and a natural language query interface powered by a Retrieval-Augmented Generation (RAG) architecture.

Instead of just looking at charts, a user can ask plain-English questions like *"What should the restaurant do to increase revenue?"* and receive data-driven insights and recommendations generated by Google Gemini.

---

## Project Overview

This project simulates a real-world business analytics solution for a restaurant operation. It answers key business questions using SQL, visualizes performance trends in an interactive dashboard, and enables natural language querying of the data through an AI assistant.

**Business questions answered:**
- Which days and time periods generate the most revenue?
- What is the average tip percentage across customer segments?
- How does party size impact total revenue?
- Do smoking section customers tip differently?
- How does revenue and tipping behavior differ by gender?
- What operational changes would improve revenue performance?

## RAG Pipeline

The AI assistant was built using a Retrieval-Augmented Generation (RAG) architecture:

1. **Load** — The restaurant dataset (CSV) is loaded into the pipeline
2. **Chunk** — Data rows are converted into text chunks for embedding
3. **Embed** — Google Gemini generates vector embeddings for each chunk
4. **Store** — Embeddings are stored in a ChromaDB vector database
5. **Query** — User submits a natural language question
6. **Retrieve** — ChromaDB retrieves the most relevant data chunks
7. **Generate** — Google Gemini synthesizes a plain-English answer with insights and recommendations

**Example queries the assistant can answer:**
- *"What should the restaurant do to increase revenue?"*
- *"What are the key revenue drivers in this project?"*
- *"When should we schedule more staff?"*
- *"Which customer segments are most valuable?"*

## SQL Queries

All queries were run in Google BigQuery. Replace `your_project.your_dataset.tips_data` with your own BigQuery table reference.

### 1. Preview Data
```sql
SELECT *
FROM `your_project.your_dataset.tips_data`
LIMIT 10;
```

### 2. Revenue by Day
```sql
SELECT `day`,
       SUM(total_bill) AS total_revenue
FROM `your_project.your_dataset.tips_data`
GROUP BY `day`
ORDER BY total_revenue DESC;
```

### 3. Revenue by Time (Lunch vs Dinner)
```sql
SELECT `time`,
       SUM(total_bill) AS total_revenue
FROM `your_project.your_dataset.tips_data`
GROUP BY `time`
ORDER BY total_revenue DESC;
```

### 4. Average Tip Percentage
```sql
SELECT ROUND(AVG(tip / total_bill) * 100, 2) AS avg_tip_percent
FROM `your_project.your_dataset.tips_data`;
```

### 5. Revenue by Party Size
```sql
SELECT size,
       SUM(total_bill) AS total_revenue
FROM `your_project.your_dataset.tips_data`
GROUP BY size
ORDER BY size;
```

### 6. Revenue by Day + Time (Combined Insight)
```sql
SELECT `day`,
       `time`,
       SUM(total_bill) AS total_revenue
FROM `your_project.your_dataset.tips_data`
GROUP BY `day`, `time`
ORDER BY total_revenue DESC;
```

### 7. Tip % by Smoker
```sql
SELECT smoker,
       ROUND(AVG(tip / total_bill) * 100, 2) AS avg_tip_percent
FROM `your_project.your_dataset.tips_data`
GROUP BY smoker;
```

### 8. Revenue + Tip % by Gender
```sql
SELECT sex,
       SUM(total_bill) AS total_revenue,
       ROUND(AVG(tip / total_bill) * 100, 2) AS avg_tip_percent
FROM `your_project.your_dataset.tips_data`
GROUP BY sex
ORDER BY total_revenue DESC;
```

## Getting Started

1. Download the dataset from the link above and upload to Google BigQuery
2. Run the SQL queries in `sql/queries.sql` against your BigQuery table
3. Open `notebooks/rag_pipeline.ipynb` in Google Colab
4. Add your Google Gemini API key to the notebook environment
5. Run all cells to launch the AI assistant
6. Ask business questions in plain English

## Key Findings

- **Dinner drives significantly more revenue than lunch** — roughly 3x more, making dinner service the primary optimization target
- **Saturday is the highest-revenue day**, with Saturday dinner being the single strongest performing segment
- **Friday is the weakest performing day** — a clear opportunity for targeted promotions or staffing adjustments
- **Larger party sizes (5-6) generate the highest total revenue** per visit
- **Smokers tip at a higher rate** than non-smokers in this dataset
